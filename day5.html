Cheatsheet Git

  1. Git config --global user.name "[Nama pengguna]" : Mengatur nama pengguna yang di gunakan untuk sebuah git di seluruh sistem secara global.
  2. Git config --global user.email "[Email pengguna]" : Mengatur email pengguna yang di gunakan untuk sebuah git di seluruh sistem secara global.
  3. Git init : Membuat repository baru atau menginisialisasi repository git didalam direktori yang ada.
  4. Git add : Menambahkan perubahan pada file ke staging area ke dalam repository git
  5. Git commit -m "[Isi Commit]" : Membuat isi commit atau merekam pesan perubahan baru pada sebuah file yang sebelumnya sudah di tambahkan ke staging area dengan perintah Git add.
  6. Git log : Menampilkan riwayat history sebuah file di dalam repository git yang telah di commit menggunakan git commit
  7. Git log --all --decorate --oneline --graph : Perintah yang digunakan untuk menampilkan riwayat komit (commit history) dalam repositori Git dengan format yang lebih ringkas dan grafis. Berikut adalah penjelasan singkat tentang opsi yang digunakan:
    • 🚦 --all : Menampilkan riwayat komit dari semua cabang (branch) dalam repositori, bukan hanya dari cabang yang saat ini Anda periksa.
    • 🚦 --decorate : Menampilkan referensi (seperti nama cabang atau tag) yang menunjuk ke setiap komit, yang bisa membantu Anda melihat hubungan antara komit dan cabang/tag yang terkait.
    • 🚦 --oneline : Menampilkan informasi komit dalam satu baris per komit, membuat tampilan lebih ringkas.
    • 🚦 --graph : Menampilkan grafik ASCII yang menunjukkan percabangan dan penggabungan (merge) dalam sejarah komit, memberikan visualisasi yang lebih baik tentang struktur repository.
  8. Git checkout : Perintah yang digunakan dalam Git untuk beberapa tujuan:
    • 🚦 Pindah antar cabang atau Branches : Anda dapat menggunakan git checkout untuk beralih dari satu cabang ke cabang lainnya dalam repositori Git Anda. Misalnya, Anda dapat menggunakan Git checkout [nama branch] untuk beralih ke cabang yang bernama [Nama branch].
    • 🚦 Mengembalikan file yang terhapus sebelumnya dengan commit : Kamu bisa menggunakan git checkout [NomorCommit5Digit] -- [namaFileYangHilang] untuk mengembalikan file yang hilang sebelumnya terakhir di commit
    • 🚦 Pindah ke filenya dengan nomor commit : Anda bisa mengubah arah headnya ke file tunggal contoh git checkout [NomorCommit5Digit].
    • 🚦 Membuat Cabang Baru : Anda juga dapat menggunakan git checkout -b [Nama Branch] untuk membuat cabang baru dan sekaligus beralih ke cabang tersebut
  9. Git remote : Perintah dalam Git yang digunakan untuk mengelola koneksi antara repositori lokal Anda dengan repositori yang berada di server jarak jauh, seperti GitHub, GitLab, atau server Git lainnya. Beberapa perintah umum yang di gunakan dengan git remote antara lain:
    • 🚦 Git remote add [nama remote] [link repository github] : Menambahkan repositori git eksternal dengan nama tertentu dan URL yang di tentukan.
    • 🚦 Git remote remove [nama remote] : Menghapus nama remote repository git
    • 🚦 Git remote -v atau Git remote show : Menampilkan daftar remote repository github yang terhubung dengan repository lokal beserta URL-nya
    • 🚦 Git remote rename [nama remote lama] [nama remote baru] : Mengganti nama remote lama dengan nama remote baru.
    • 🚦 Git remote set-url [nama remote] [https://[token generate]@github/[owner]/[Nama repository]] : Di gunakan untuk memberikan akses remote github ke direktori lokal dengan token autentifikasi
  10. Git fetch : Di gunakan untuk mengecek semua perubahan file pada repositori github.
  11. Git push : Mengirimkan perubahan file repository local ke repository remote.
  12. Git pull : Mengambil perubahan dari repositori remote dan langsung menggabungkannya ke cabang lokal yang sedang aktif. sebenarnya ini adalah kombinasi dari git fetch dan git merge.
  13. Git branch : Untuk membuat nama cabang baru sama seperti perintah "git checkout -b [nama branch]" tapi ini berbeda setelah membuat nama branch di barengi dengan perintah "git checkout [nama branch]".
  14. Git branch -d [nama branch] : Menghapus branch repository lokal bukan remote
  15. Git merge : Menggabungkan file dari satu cabang ke cabang lainnya, dan kalau misalkan terjadi conflict file maka harus di perbaiki dan file tersebut harus di commit kembali.
  16. Git rebase : Menggabungkan perubahan lebih bersih dan menghindari penggabungan berulang.
